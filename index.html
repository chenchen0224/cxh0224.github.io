我的Github+hexo个人博客是怎样炼成的

Veröffentlicht am 2017-09-29

准备工作

在搭建博客开始之前，您必须有做好以下准备工作

  • 安装了nodejs，npm，都是傻瓜式安装，此处不再赘述。
  • 安装了git for windows，git bash可打开黑窗口
  • 在GitHub上注册一个账号，并配置SSH key

安装hexo

$ npm install -g hexo


在电脑上创建一个hexo的文件夹，我的是在E盘下，E:\hexo，在Hexo文件下，右键运行Git Bash，进行初始化

$ hexo init

初始化成功后生成的一些列文件

$ hexo g //生成
$ hexo s //启动本地预览服务，打开浏览器访问 localhost:4000 即可看到内容，按Ctrl+C停止预览


执行以上命令之后，hexo就会在public文件夹生成相关html文件

修改主题

hexo官网主题
个人比较喜欢hexo-theme-next，GitHub地址，clone下来
首先下载这个主题，进入hexo\themes文件夹，右键git bash

$ git clone https://github.com/xinhua224/hexo-theme-next.git

然后在_config.yml中，进行基础配置，并将主题修改为theme: hexo-theme-next

重新执行hexo g来重新生成。

部署到Github上

在Github上new repository，新建一个仓库

clone仓库地址

在hexo文件夹下，_config.yml进行配置（注意仓库地址repository的写法，并不是简单复制），然后重新生成输入命令

$ hexo g

$ npm instal hexo-deployer-git –save，//安装hexo-deployer-git自动部署发布工具


发布到Github，输入如下命令：

$ hexo d -g


在GitHub上刷新，public上的文件就发布上来了

测试访问，在浏览器打开xinhua224.github.io

写博客

创建新文章，git bash输入

$ hexo new 我的Github+hexo个人博客是怎样炼成的


在本地E:\hexo\source_posts目录下就新生成了新博客页的md文件

在sublime中打开该文件，用markdown语法书写即可。
写博客时，$ hexo s启动本地服务，在http://localhost:4000/，查看本地效果。

写完后发布更新到GitHub上
重新生成输入命令

$ hexo g

更新到Github，输入如下命令：

$ hexo d -g
